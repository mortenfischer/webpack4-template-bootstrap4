Feature description

What follows is a description of what this particular configuration of Webpack does for you.

Localhost dev environment and Production build script

Start dev environment: npm run start.

Compile to production: npm run build.

Multiple entry files

You can have as many entry files as you like. Each entry will result in a new destination file. You can see the config here in config/webpack.common.js.

Please note that Webpack outputs JS files for everything, also CSS. You should however ignore those and only mind the CSS files.

SCSS transpilation

You will get automatic transpilation from .scss files into .css files. This means you have the posibility to tap into the power of SCSS/Sass.

ES6+ transpilation

All .js files will be transpiled to match ES5 that works in all browsers, including Internet Explorer 11 specifically. In a few years this transpilation can theoretically be removed, but as with most things fronted: It depends!

Automatic copying of css/js files to your solutions static file folders

Currently all files that are transpiled when you call npm run build end up in the "build" folder. Here they are not of much use, and it is not recommended to configure webpack to use destinationfolders that are not "local" to your frontend solution.

To enable automatic copying of files (so you dont have to manually move them everytime) you need to look at the config file called config/copy-to-solution.js.

In this config file you specify the path to your final destination. The copying will run AFTER all transpilation have completed, as can be seen in config/webpack.prod.js.
